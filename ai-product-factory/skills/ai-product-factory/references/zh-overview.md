# 中文说明

> 本文件只用于帮助中文用户理解流程。英文 `SKILL.md`、英文 reference、Schema、模板和脚本是唯一执行依据；如有差异，以英文为准。

这个 Skill 是总编排器，适合“从想法到上线”“继续上次产品流程”或需要统一质量门槛的任务。

它会：

- 判断项目属于新产品、已有产品增强还是无 UI 产品。
- 从状态文件恢复，不把“文件存在”误认为“已经批准”。
- 根据风险与不确定性选择 Lite、Standard 或 Deep。
- 每次只调用当前阶段对应的一个 Worker。
- 校验产物、请求明确审批、记录版本和事件。
- 上游变更时让受影响的下游产物失效并重新评审。

职责边界：

- 用户批准范围、重大取舍、发布和生产部署。
- Codex 负责产品、架构、计划、编排、评审和质量门槛。
- OpenDesign 负责 UI/UX 设计。
- Claude Code 负责代码实现。

中文请求可以用中文输出，但状态值、ID、路径和标题保持英文，执行时不会加载一套重复的中文规则。
