# 中文说明

> 本文件只用于中文理解。英文 `SKILL.md`、`delivery-policy.md` 和模板是唯一执行依据；冲突时以英文为准。

这个 Skill 负责把已批准的任务交给 Claude Code 实现，然后由 Codex 独立验证、评审并管理发布门槛。

关键原则：

- Claude Code 只实现已批准的 `TASK-`，不能自行改需求或架构。
- Codex 检查 diff、测试和命令结果，不把“已经实现”的口头说明当作证据。
- lint、test、build 以及 TDD 要求的专项检查必须有真实结果。
- Critical 和 Major 问题解决前不能通过评审。
- 发布前必须确认版本、提交、迁移、配置、监控、冒烟和回滚。
- 生产部署需要用户明确批准，部署启动不等于完成。

中文报告可以自然表达，但状态、ID、检查结果和决策值保持英文。
