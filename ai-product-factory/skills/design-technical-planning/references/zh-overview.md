# 中文说明

> 本文件只是中文导读。英文 `SKILL.md`、两个英文 policy 和模板是唯一执行依据；冲突时以英文为准。

这个 Skill 负责把已批准的 PRD 转换为设计和研发可以执行的合同，但不写生产代码。

它覆盖四类产物：

- `UI_INPUT.md`：给 OpenDesign 的用户流程、页面、状态和约束。
- `UI.md`：对 OpenDesign 原型进行覆盖检查后形成实现规范。
- `TDD.md`：基于真实仓库制定架构、数据、API、安全、测试和回滚。
- `sprint-001.md`：把工作拆成 Claude Code 可以逐项实现和验证的任务。

无 UI 产品不会伪造页面，而是正式标记 UI 产物不适用。

如果设计与 PRD 冲突、技术方案需要重大取舍，或上游版本未批准，会退回相应阶段，不会在下游文档里偷偷改需求。
