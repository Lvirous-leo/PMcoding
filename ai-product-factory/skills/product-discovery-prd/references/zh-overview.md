# 中文说明

> 本文件只做中文解释。英文 `SKILL.md`、`analysis-method.md`、`evidence-policy.md` 和 PRD 模板是唯一执行依据；冲突时以英文为准。

这个 Skill 用于需求分析、产品调研、竞品与用户问题研究、PRD，以及 GO / PIVOT / KILL / HOLD 决策。

核心不是把用户说的功能直接写进 PRD，而是先判断：

- 谁在什么场景遇到什么问题。
- 真正想改变的结果是什么。
- 当前证据是事实、推断、假设还是未知。
- 最小方案能否形成业务、场景和逻辑闭环。
- 对其他角色、流程、数据和指标有什么系统影响。
- 是否应该简化、配置化或暂时保持人工。
- 哪个关键假设最值得先验证。

证据从 L0 到 L3 逐步增强。风险越高、越不可逆，GO 所需证据越高；关键假设未解决时不能建议 GO。

中文 PRD 正文是允许的，但 `REQ-001`、`AC-001`、L0–L3 和决策值保持英文。
